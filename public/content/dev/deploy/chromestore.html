Chrome web store deploy

Description

The Chrome web store deploy lets you package your application for the Chrome web store.
The Chrome web store is a marketplace of web applications that can be installed on the Chrome browser. These applications can use additional features that traditional websites cannot use, for example, running offline or being installed locally.

The packaged application generated by the Joshfire Factory doesn't require a web server: your application will be downloaded and executed on your user's Chrome browser.

Configuration

It is recommanded to provide an icon. To do so, upload an icon to your application in the configuration tab.

Description: A text that describes briefly the application. You will be able to provide a longer description on the Chrome web app submission page. It should be plain text, no HTML or other formatting. And shouldn't be longer than 132 characters.

Version: a version number of your application. This version number should be increased at every new submission on the Chrome web store.

Testing on Google Chrome

The Chrome browser has built-in support to test your application localy before submitting to the Chrome web store. This is done on the chrome://extensions/ page (also accessible via menus), and by clicking on “Developer Mode”, you get access to tools for adding and reloading your application. You will need to unzip the generated zip file in a folder and then to select this folder after clicking on the "Load unpacked extension" button.

Submitting your application to the Chrome web store

You need to use the Chrome Developer Dashboard to publish your application. If you haven't already, you will be asked to pay a small fee of 5$ to open your account.
The generated .zip file can then directly be uploaded to the application submission form on the Chrome Developer Dashboard.
