So, your template is ready to ship? Good! Now it's time to take a decision about the visibility of your template.

Keep a template private

If you want to keep your template private, use the Create new button in the left pane of the templates listing page. This functionality allows you to import your template from any (private) git repository. If you followed the instructions in the Bootstrap script during template development section in the Create template entry point(s) chapter, then simply use the Update from git button on the details page of your private template to refresh the code when needed. If not, please note that the import works provided that:

  • the manifest file is correct;
  • the files the manifest links to (starting points in particular) exist;
  • the git repository is either public or accessible by the factory (in other words, you need to add the public SSH key generated by the factory and shown on the import page to the list of SSH keys that can access the private repository).

Publish a template

The factory periodically refreshes the list of templates available based on the repositories of the dedicated Joshfire-platform-templates organization on github. Publishing your template is thus a matter of ensuring your template becomes a repository of that organization.

This operation is manual as of today. Please get in touch at contact@joshfire.com to submit your template. Before you submit your template, ensure that you have thoroughly tested it as a factory template using the private method explained above.
