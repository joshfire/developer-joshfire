Bind to datasources

The platform is focused on the generation of content-based Web applications. Through the user interface, users select the feeds that they would like to expose in the application. Templates fetch and render data from the datasources.

Define datasource inputs

Again, pay attention to the title and description of the datasource input to guide users when they select datasources. The description should provide an indication of the expected type of data (e.g. pictures? videos? blog post?) and clarifies how the template will consume that information.

The description should also highlight the maximum number of data feeds handled by the template for datasources that support multiple data feeds (i.e. when the multiple option is set).
