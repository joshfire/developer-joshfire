Getting started with templates

What is a template?

Templates play a central role in the factory. Templates impose the layout and functionality of the generated Web application. They impose the types and number of datasources that the user can feed the Web application and what the Web application does with it. Templates also handle user interactions.

Templates may be very different, from generic templates that handle all types of data and define a cross-device interface to specialized templates that focus on rendering a picture feed on TV sets. Templates address a specific user need.

Why develop a template?

You may wish to develop a template for your own private use, for a client project, to sell it to Web applications developers through the factory or for fame. The factory allows you to do any of these, providing the market place for Web application developers and the integration with datasources of all kinds.

In particular, the visibility of the template you develop is entirely up to you. You may decide to keep your template private and thus be the only one who can see it and use it to build applications, or you may publish the template to the factory for use by others, for a given fee or free of charges.

Anatomy of a template

A template consists of:

  • A manifest file named package.json that describes the properties of the template. The factory uses the information contained in the manifest file to adjust the actions proposed through the user interface. The information exposed to the generated Web application also depends on the contents of the manifest file. The name and location of the manifest file cannot be changed. It must be package.json and the file must appear in the root folder of the template.
  • One or more index file(s) that serves as entry point for the Web application (index.html in the same folder as the manifest file is assumed unless overridden in manifest file). These entry points must be valid HTML5 files.
  • A set of resources (images, videos, JavaScript code, HTML pages, etc.)
  • The JavaScript logic of the Web application.

A template runs on client devices and cannot include code that would require server-side execution.

Datasources and configuration settings selected by users are exposed to the template through a Joshfire.factory global object.

A minimal template

A minimal "Hello world!" template consists of two files in the same folder:

  • an index.html file that contains markup such as: 
    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>Hello World</title>
  </head>
  <body>
    <p>Hi, there.</p>
  </body>
</html>
  • a package.json manifest file that contains the following code: 
    {
  "name": "hello-world",
  "fullName": "Hello World!",
  "version": "0.0.1",
  "description": "Hello world minimal template. Displays a static 'Hello World!' message.",
  "joshfire": {
    "images": {
      "small": "http://i.imgur.com/sm0MA.png",
      "big": "http://i.imgur.com/x5NbC.png"
    }
  }
}

Such a minimal template does not support any datasource input and does not encapsulate any JavaScript logic. This example is essentially dull. The templates documentation explains how to create something more useful!
