Propose more configuration settings

Templates may customize the look-and-feel and functionality of the generated Web application based on additional configuration settings. For instance, a template that features recipes of one's grand-mother may ask for the nickname and a picture of the said grand-mother.

Define additional configuration settings

The options section in the manifest file specifies additional parameters in the schema subsection and visual hints for the factory for form rendering purpose in the form subsection.

The following manifest code would allow to define a template for the recipes example mentioned above:

{
  "name": "grandma-recipes",
  "fullName": "Grandma's recipes",
  "version": "0.0.1",
  "description": "Ideal to list the recipes of your Grandma."
  "datasources": {
    "recipes": {
      "title": "Recipes",
      "description": "The list of recipes to include.",
    }
  }
  "options": {
    "schema": {
      "nickname": {
        "type": "string",
        "title": "Grandma's nickname"
      },
      "picture": {
        "type": "string",
        "title": "A picture of Grandma"
      }
    },
    "form": {
      "nickname",
      "picture",
      "datasource.recipes"
    }
  }
}

Retrieve configuration settings

From the template's code, configuration settings are exposed in the Joshfire.factory.config.template.options object.

For instance, to access Grandma's nickname and picture defined above from the template's code:

var options = Joshfire.factory.config.template.options;
var grandma = {};
grandma.nickname = options.nickname;
grandma.picture = options.picture;
