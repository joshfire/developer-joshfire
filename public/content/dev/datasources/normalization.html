Data normalization

The whole purpose of a datasource is to expose content to templates following a structure that templates may easily consume. Factory templates have no idea about the origin of a particular item they receive. All they see is an object with properties. That is on purpose. Basically, if the object exposes the properties that the template understand, the item can be used. If not, the template is likely going to ignore the item.

Using schema.org

Finding the right level of abstraction and the right set of properties to describe things is not an easy task. The goal of the Factory is also to integrate easily with data on the Web. Many vocabularies are used on the Web to represent specific verticals of data. The Factory uses the schema.org type hierarchy for core objects returned by datasources.

Item types and properties

Check the Datasource item type reference for more information about the item types and properties returned by existing datasources.

It is good practice to set basic properties of the Thing type whenever possible:

  • The name property should contain the content's title
  • The image property should contain an image to use as thumbnail for the content
  • The description property should be a short raw string that summarizes what the content is about
  • The url property should contain a GUID for the content
