Import a private template

The Joshfire Factory lets you import your own templates to build custom applications. Imported templates are private to you, other Factory users will not see them.
Your imported private templates can then be found in the template tab, by selecting the private template filter in the sidebar.
To learn more about templates development, read the template documentation page

There are two ways to import your custom template into the Factory: zip upload or git import. If you are not familiar with git, we suggest that you use the zip upload option.

zip upload

Simply package your template code in a .zip archive and upload it into the Factory. Our system will look for a package.json description file, if it doesn't exist, it will look for a start page named index.html. Then, your template is ready to be used in your application.

Updating the code

To update your template code, create a new .zip archive on your computer. Then go to your private template page, click on the "Update via archive upload" button on the right and upload your new archive.
Your template code should then be updated. However, applications using this template still run the previous version. You have to go to the Configuration tab and then visit the Updates menu.

git import

The git import method is for advanced users. It requires to use the version control software called git.
Using git import requires you to have your package.json manifest file or your index.html file at the root of the repository.
The latest commit of the master branch will be imported as a private template.

Importing a public repository

Public repository doesn't require authentification. For public repositories, enter the git read-only URL or the public HTTP address of your repository. For instance:

  • git://github.com/joshfire-platform-templates-test/good-repo.git
  • https://bitbucket.org/joshfire/template-portfolio.git

Importing a private repository

A private repository cannot be accessed by the Factory without giving it the rights to do so. We generate an SSH key that you need to add to your private repository.
For private repositories, add the SSH key that appears on the page to your git repository so that the factory can access it. On GitHub, you can do this from the "Admin / Deploy Keys" menu. On BitBucket, check the "Account / SSH Keys" menu. Then enter the SSH address of your repository. For instance:

  • git@github.com:joshfire-platform-templates-test/good-repo.git
  • git@bitbucket.org:joshfireplatformtest/template-private.git

Updating the code

To update your template code, push your latest commits to the master branch of your git repository. Then go to the page of your private template, and click on the "Update from git" button on the right. The latest code from your git repository will be pulled by the Factory.
Your template code should then be updated. However, applications using this template still run the previous version. You have to go to the Configuration tab and then visit the Updates menu.
